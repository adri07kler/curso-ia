5.4 Revisión de código generado

Módulo 5 · Video

Objetivo: Aprender técnicas de code review específicas para código generado por IA.

Proceso de revisión

  1. Primera pasada — Estructural: ¿Sigue los patrones del proyecto? ¿Respeta convenciones?
  2. Segunda pasada — Lógica: ¿La lógica es correcta? ¿Faltan condiciones? ¿Los bucles terminan?
  3. Tercera pasada — Seguridad: ¿Validación de entrada? ¿Sanitización? ¿Credenciales?
  4. Cuarta pasada — Mantenibilidad: ¿Será fácil de modificar? ¿Funciones con una sola responsabilidad?

Señales de alerta específicas de IA

  • Funciones "comodín" que hacen demasiado
  • Código muerto (variables no usadas)
  • Dependencias que no existen
  • Código que parece de otro lenguaje
  • Comentarios genéricos ("# Add your code here")
