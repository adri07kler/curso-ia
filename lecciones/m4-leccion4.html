4.4 Cómo pedir correcciones a la IA

Módulo 4 · Video

Objetivo: Aprender a pedir correcciones efectivas cuando la IA da una respuesta incorrecta.

Estrategias

  1. Sé específico sobre qué está mal: "La línea 15 maneja el caso de éxito pero no el caso de error"
  2. Proporciona el error exacto: "Lanza TypeError: Cannot read property 'id' of undefined"
  3. Pide alternativas, no solo fixes: "Dame 3 opciones para manejar este caso borde"
  4. Rechaza respuestas enteras si es necesario
  5. Añade al contexto lo que aprendiste para que no vuelva a pasar

Errores comunes

  • Ser vago ("arréglalo")
  • No decir qué esperas en su lugar
  • Aceptar la primera corrección sin revisarla

Introducción

La IA no siempre acierta a la primera. Saber cómo pedir correcciones efectivas es una habilidad crítica en el desarrollo asistido. En esta lección aprenderás estrategias concretas para comunicar errores a la IA y obtener soluciones precisas, ahorrando tiempo y evitando frustraciones.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Identificar qué información incluir al reportar un error a la IA.
  • Aplicar estrategias de corrección específicas según el tipo de error.
  • Pedir alternativas múltiples para comparar soluciones.
  • Rechazar respuestas incorrectas de manera constructiva.
  • Alimentar el contexto de la IA para evitar errores recurrentes.

Desarrollo del contenido

La comunicación efectiva con la IA

Cuando la IA comete un error, la tentación es decir "arréglalo" y esperar un milagro. Sin embargo, la IA no "entiende" el error como un humano; necesita información estructurada para corregirse. El principio fundamental es: entre más específico seas sobre qué está mal, mejor será la corrección.

Tipos de errores y cómo reportarlos

  • Error de lógica: "La función devuelve el valor incorrecto cuando la entrada es negativa. El caso de prueba entrada=-1 debería devolver 0 pero devuelve -1."
  • Error de sintaxis: "La línea 15 lanza SyntaxError: unexpected EOF while parsing. Revisa paréntesis y bloques."
  • Error de seguridad: "Este código es vulnerable a SQL injection en la línea 23. Necesito una versión con consultas parametrizadas."
  • Error de rendimiento: "La función hace una consulta a la BD por cada elemento del array. Necesito una solución que haga una sola consulta."

El ciclo de corrección efectiva

  1. Especifica el problema: Describe qué está mal, dónde ocurre y cuál es el comportamiento esperado.
  2. Proporciona contexto: Incluye el código relevante, el error exacto y las entradas que lo provocan.
  3. Solicita una solución: Sé claro sobre qué tipo de corrección necesitas.
  4. Evalúa la respuesta: Revisa si la corrección resuelve el problema sin introducir nuevos errores.
  5. Itera si es necesario: Si la corrección no es suficiente, proporciona retroalimentación adicional.

Ejemplos

Ejemplo 1: Corrección de error en API

Prompt original: "Crea un endpoint POST /usuarios"

Respuesta incorrecta: Código que no valida el email duplicado.

Corrección efectiva:

"El endpoint POST /usuarios que generaste no maneja el caso de email duplicado. Cuando intento registrar un email existente, devuelve 500 en lugar de 409. Añade una verificación antes de insertar: si el email existe, devuelve {'error': 'Email ya registrado'} con código 409."

Ejemplo 2: Corrección de componente React

Prompt original: "Crea un componente de contador en React"

Respuesta incorrecta: El botón de decremento permite valores negativos.

Corrección efectiva:

"El contador no debe permitir valores negativos. Modifica la función decrement para que valide que count > 0 antes de restar. Si count es 0, el botón debe estar deshabilitado."

Ejemplo 3: Corrección de prompt engineering

Prompt original: "Genera un resumen de este artículo"

Respuesta incorrecta: Resumen muy genérico de 3 líneas.

Corrección efectiva:

"El resumen es demasiado corto y genérico. Necesito un resumen estructurado con: (1) idea principal en 2 líneas, (2) 3 puntos clave del desarrollo, (3) conclusión. Mínimo 150 palabras."

💡 Consejo del instructor: Cuando recibas una respuesta incorrecta de la IA, antes de pedir una corrección, pregúntate: "¿Qué información necesita la IA para entender exactamente qué está mal?" Proporciona esa información en tu primer mensaje de corrección.
⚠ Errores comunes:
  • Aceptar la primera corrección sin revisar: La IA puede "arreglar" un error e introducir dos nuevos.
  • No proporcionar el mensaje de error exacto: "No funciona" no ayuda. El error exacto es la clave.
  • Corregir en el idioma equivocado: Si el código está en español pero pides la corrección en inglés, la IA puede mezclar estilos.

Buenas prácticas

  • ✔ Incluye el mensaje de error completo (stack trace) cuando sea posible.
  • ✔ Señala la línea o función específica donde ocurre el error.
  • ✔ Describe el comportamiento esperado, no solo el incorrecto.
  • ✔ Pide alternativas cuando el problema tiene múltiples soluciones válidas.
  • ✔ Verifica cada corrección antes de continuar con el siguiente paso.
  • ✔ Si la IA insiste en el mismo error, reformula el problema desde cero.

Actividad práctica

Escribe 3 prompts incorrectos a propósito y pide a una IA que genere código para cada uno. Luego, aplica las estrategias de corrección aprendidas para obtener la versión correcta. Documenta el proceso completo: prompt original, respuesta incorrecta, corrección aplicada y resultado final.

Mini reto

Diseña una plantilla reutilizable de "corrección estructurada" que puedas copiar y pegar cada vez que necesites pedir una corrección a la IA. La plantilla debe incluir campos para: contexto, problema, ubicación, comportamiento esperado y ejemplo. Comparte tu plantilla.

Resumen

  • Ser específico sobre qué está mal acelera la corrección por parte de la IA.
  • Incluir el error exacto y el contexto es fundamental para obtener una solución precisa.
  • Pedir alternativas permite comparar y elegir la mejor solución.
  • Cada corrección debe ser revisada antes de integrarse.
  • Alimentar el contexto con lo aprendido evita errores recurrentes.
