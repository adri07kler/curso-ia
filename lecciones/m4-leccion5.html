4.5 Cómo revisar una respuesta de código

Módulo 4 · Documento

Objetivo: Proporcionar una guía sistemática para revisar código generado por IA.

Checklist de revisión

  1. ¿Cumple el requerimiento? ¿La función hace exactamente lo que se pidió?
  2. ¿Respeta el alcance? ¿Hace algo que no se le pidió?
  3. ¿Maneja errores? ¿Qué pasa si la entrada es inválida?
  4. ¿Es legible? ¿Nombres claros? ¿Estructura lógica?
  5. ¿Introduce dependencias innecesarias?
  6. ¿Es seguro? ¿SQL injection, XSS, exposición de datos?
  7. ¿Tiene pruebas?
  8. ¿Se entiende por qué se hizo así?

Introducción

Revisar código generado por IA es una habilidad diferente a revisar código escrito por humanos. La IA puede producir código sintácticamente perfecto pero lógicamente incorrecto, inseguro o ineficiente. En esta lección desarrollarás un proceso sistemático de revisión que te permitirá detectar problemas antes de que lleguen a producción.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Aplicar una checklist sistemática para revisar código generado por IA.
  • Identificar problemas comunes como falta de manejo de errores, dependencias innecesarias y vulnerabilidades de seguridad.
  • Evaluar si el código cumple el requerimiento sin exceder el alcance.
  • Determinar cuándo el código generado es aceptable y cuándo debe rechazarse.
  • Documentar hallazgos de la revisión para retroalimentar a la IA.

Desarrollo del contenido

¿Por qué la revisión de código IA es diferente?

El código generado por IA suele ser correcto en forma pero puede fallar en fondo. La IA no tiene contexto del negocio, no conoce la arquitectura completa del proyecto y puede alucinar APIs o funciones que no existen. Por eso, una revisión superficial no es suficiente.

Los 8 puntos de la checklist explicados

  • 1. Cumplimiento del requerimiento: Verifica que la función haga exactamente lo que pediste. Ni más, ni menos. La IA a veces interpreta incorrectamente o añade funcionalidades no solicitadas.
  • 2. Alcance: La IA puede generar código adicional no solicitado (ej: añadir logging, crear rutas extra, modificar archivos existentes). Esto puede introducir comportamientos inesperados.
  • 3. Manejo de errores: El código generado frecuentemente asume condiciones ideales. Verifica qué pasa con entradas vacías, nulas, mal formateadas o fuera de rango.
  • 4. Legibilidad: Nombres de variables claros, estilo consistente, estructura lógica. Si no se entiende a simple vista, probablemente tenga problemas.
  • 5. Dependencias innecesarias: La IA tiende a usar librerías externas para tareas simples. Revisa si puedes lograr lo mismo con el estándar del lenguaje.
  • 6. Seguridad: SQL injection, XSS, exposición de datos sensibles, falta de autenticación. Este es el punto más crítico y el que más se pasa por alto.
  • 7. Pruebas: El código generado debe incluir o al menos permitir pruebas. Si la IA no las generó, evalúa qué pruebas necesitarías escribir.
  • 8. Trazabilidad: ¿Entiendes por qué se tomó cada decisión? Si el código es un "bloque mágico" que no comprendes completamente, es una señal de alerta.

Ejemplos

Ejemplo 1: Endpoint sin manejo de errores

Código generado:

app.get('/usuario/:id', (req, res) => {
  const usuario = db.query(`SELECT * FROM usuarios WHERE id = ${req.params.id}`);
  res.json(usuario);
});

Problemas detectados:

  • 🚨 SQL injection: el parámetro se concatena directamente.
  • ⚠ Sin manejo de errores: si el query falla o el usuario no existe, la respuesta es impredecible.
  • ⚠ Sin validación: el id podría no ser un número.

Ejemplo 2: Función con dependencia innecesaria

Código generado:

import _ from 'lodash';
const sumArray = (arr) => _.sum(arr);

Problema: Se importa lodash completo para una operación que JS nativo resuelve con reduce.

Versión mejorada:

const sumArray = (arr) => arr.reduce((a, b) => a + b, 0);

Esto elimina la dependencia externa y hace el código más ligero.

Comparación: Código aceptable vs. Rechazable

CriterioAceptableRechazable
CumplimientoSatisface el requerimientoNo hace lo que se pidió
SeguridadSin vulnerabilidades identificadasInyección SQL, XSS, datos expuestos
Manejo de erroresCasos borde cubiertosSin validación ni try/catch
RendimientoComplejidad adecuada al contextoO(n²) cuando podría ser O(n)
DependenciasUsa solo lo necesarioLibrerías externas para tareas triviales
LegibilidadNombres claros, lógica evidenteCódigo ofuscado o sobreingeniería

Diagrama: Proceso de revisión sistemática

1
Lee el código completo
2
Verifica contra checklist
3
Identifica hallazgos
4
Clasifica: crítico / mejorable / ok
5
Decide: aceptar / corregir / rechazar
💡 Consejo del instructor: Crea una plantilla de revisión que puedas reutilizar. Un documento con los 8 puntos de la checklist y espacio para notas por cada punto. Revisar con una plantilla asegura que no olvides ningún aspecto crítico.
⚠ Errores comunes:
  • Confiar en que "si compila, funciona": El código puede compilar perfectamente pero fallar en producción por casos borde no contemplados.
  • No revisar dependencias: La IA puede agregar librerías con vulnerabilidades conocidas.
  • Pasar por alto la seguridad: Es el aspecto más omitido en revisiones de código generado por IA.

Buenas prácticas

  • ✔ Usa la checklist de 8 puntos en cada revisión, sin excepción.
  • ✔ Clasifica los hallazgos en críticos, mejorables y aceptables.
  • ✔ No aceptes código que no entiendas completamente.
  • ✔ Prueba el código en un entorno aislado antes de integrarlo.
  • ✔ Documenta los hallazgos para compartir con el equipo.
  • ✔ Si encuentras el mismo tipo de error repetidamente, ajusta tus prompts.

Actividad práctica

Toma 3 fragmentos de código generados por IA (pueden ser de lecciones anteriores o generados ahora) y aplícales la checklist de 8 puntos. Documenta para cada uno: qué puntos cumple, qué puntos falla, y tu decisión final (aceptar, corregir o rechazar). Incluye capturas o el código revisado.

Mini reto

Crea una herramienta web simple (HTML+JS) que implemente la checklist de revisión interactiva. El usuario debe poder seleccionar qué puntos cumple el código, agregar notas por punto, y recibir una recomendación automática (aceptar / corregir / rechazar) basada en sus respuestas.

Resumen

  • Revisar código de IA requiere un enfoque sistemático; la checklist de 8 puntos es tu mejor aliada.
  • Los problemas más frecuentes son: falta de manejo de errores, dependencias innecesarias y vulnerabilidades de seguridad.
  • Clasificar hallazgos en crítico/mejorable/ok ayuda a priorizar.
  • Si no entiendes el código, no lo aceptes.
  • La revisión debe hacerse siempre antes de integrar, nunca después.
