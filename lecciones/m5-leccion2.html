5.2 Pruebas mínimas necesarias

Módulo 5 · Documento

Objetivo: Definir el conjunto mínimo de pruebas para validar código generado por IA.

Introducción

Cuando un desarrollador escribe código manualmente, el proceso de pensar en casos de prueba ocurre de forma natural durante la implementación. Con código generado por IA, este paso se omite: la IA produce una solución y el desarrollador tiende a aceptarla sin cuestionarla. Establecer un conjunto mínimo de pruebas obligatorias —smoke test, caso feliz, caso borde, error y regresión— es la red de seguridad que separa un borrador de IA de código listo para producción.

Esta lección te dará una checklist práctica que puedes aplicar en menos de 5 minutos por función generada, reduciendo drásticamente la probabilidad de bugs.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Definir los 5 tipos de pruebas mínimas para código generado por IA
  • Escribir pruebas de humo, caso feliz, caso borde, error y regresión
  • Aplicar la regla de "al menos una prueba de caso feliz y una de caso borde o error"
  • Identificar qué tipo de prueba cubre cada posible fallo en código de IA

Tipos de pruebas mínimas

TipoDescripciónEjemplo
Prueba de humoVerifica que se ejecuta sin erroresLlamar con datos válidos
Prueba de caso felizVerifica el resultado esperadosuma_pares([2,4,6]) == 12
Prueba de caso bordeVerifica límitessuma_pares([]) == 0
Prueba de errorVerifica fallo correctosuma_pares("texto") → TypeError
Prueba de regresiónNo rompe nada existenteEjecutar tests existentes

Ejemplo mínimo

def test_suma_pares_caso_feliz():
    assert suma_pares([1, 2, 3, 4]) == 6

def test_suma_pares_lista_vacia():
    assert suma_pares([]) == 0

def test_suma_pares_error_tipo():
    with pytest.raises(TypeError):
        suma_pares("no es lista")

Explicación del ejemplo

La prueba de humo está implícita en la de caso feliz (si se ejecuta sin error, pasa humo). El caso borde (lista vacía) verifica que la función no falle con entrada límite. La prueba de error asegura que la función rechace entradas inválidas. Con estas 3 pruebas cubrimos los 5 tipos mínimos.

Diagrama: Pirámide de pruebas mínimas

🥇 Prueba de humo — ¿Ejecuta sin errores?
🥇 Caso feliz — ¿Resultado correcto?
⚠️ Caso borde — ¿Maneja límites?
❌ Prueba de error — ¿Falla correctamente?
🔄 Prueba de regresión — ¿Sigue funcionando?

Cada función generada por IA necesita al menos: caso feliz + caso borde o error

Consejo del instructor: No necesitas un framework de testing elaborado. Un simple assert en Python o console.assert en JavaScript ya es mejor que nada. La meta es tener al menos 2 pruebas por función generada: caso feliz + caso borde o error.

Errores frecuentes

  • Solo probar el caso feliz: Es la prueba más común pero la menos útil sola. Sin casos borde, no sabes si la función maneja entradas límite.
  • Confiar en que "si compila, funciona": El código sintácticamente válido puede ser semánticamente erróneo. Las pruebas verifican comportamiento, no sintaxis.
  • Escribir pruebas después de implementar en producción: Las pruebas deben hacerse antes de integrar el código, no cuando ya está desplegado.
  • Pruebas que prueban la implementación, no el comportamiento: Probar que se llamó a una función interna no es lo mismo que probar que el resultado es correcto.

Buenas prácticas

  • Escribir pruebas antes de integrar: Genera el código con IA, escribe las pruebas, verifica que pasen
  • Una prueba por tipo de fallo posible: Cada tipo de entrada incorrecta merece su propia prueba
  • Automatizar las pruebas: Integra las pruebas en un pipeline CI/CD para que se ejecuten en cada cambio
  • Documentar qué prueba cada caso: Usa nombres descriptivos como test_suma_pares_lista_vacia
  • Re-ejecutar pruebas al regenerar: Si pides a la IA que modifique el código, ejecuta todas las pruebas de nuevo

Actividad práctica

La IA ha generado esta función que debe contar cuántas vocales tiene una cadena:

def contar_vocales(texto):
    return sum(1 for c in texto if c in "aeiou")

Escribe las pruebas mínimas (humo, caso feliz, caso borde y error) para esta función. Considera:

  1. ¿Qué pasa si el texto está vacío?
  2. ¿Qué pasa si no hay vocales?
  3. ¿Qué pasa si hay mayúsculas?
  4. ¿Qué pasa si el argumento no es string?
Solución: La función falla con mayúsculas ("AEIOU" no contiene "aeiou"). Debería usarse c.lower() in "aeiou". Además, la prueba de error debe verificar que entrada numérica lance TypeError o se maneje adecuadamente.

Mini reto

Genera con IA una función que encuentre el número más frecuente en una lista. Sin revisar el código, escribe las 5 pruebas mínimas (humo, caso feliz, caso borde, error, regresión). Luego ejecuta el código generado contra tus pruebas. ¿Cuántas pruebas fallaron? Ese es el número de bugs que la IA introdujo.

Resumen

  • Las 5 pruebas mínimas son: humo, caso feliz, caso borde, error y regresión
  • Por cada función generada por IA debe haber al menos una prueba de caso feliz y una de caso borde o error
  • Las pruebas son la red de seguridad que detecta bugs antes de producción
  • El tiempo invertido en pruebas se recupera evitando incidentes en producción
