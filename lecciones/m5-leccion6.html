5.6 Auditar una solución generada por IA

Módulo 5 · Ejercicio

Objetivo: Practicar la auditoría completa de una solución generada por IA.

Introducción

Has aprendido los riesgos de confiar ciegamente en la IA, los tipos de pruebas mínimas, cómo identificar casos borde y cómo revisar código sistemáticamente. Ahora es momento de poner todo junto: este ejercicio te presenta una solución generada por IA que debes auditar como si fuera parte de un merge request real en tu trabajo. Identificarás problemas, propondrás correcciones y decidirás si el código está listo para producción.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Auditar una solución completa generada por IA identificando problemas múltiples
  • Diferenciar entre problemas críticos (bloqueantes) y problemas menores (mejoras)
  • Escribir las pruebas mínimas que validen la corrección de una función generada por IA
  • Tomar una decisión informada de aprobar, requerir cambios o rechazar código generado por IA

Enunciado

Requerimiento: Función que recibe un archivo CSV y devuelve un JSON con el promedio de cada columna numérica.

Solución generada:

import csv, json

def csv_to_json_promedios(ruta_archivo):
    datos = []
    with open(ruta_archivo, 'r') as archivo:
        lector = csv.DictReader(archivo)
        for fila in lector:
            datos.append(fila)
    promedios = {}
    for columna in datos[0].keys():
        suma = 0; count = 0
        for fila in datos:
            try:
                suma += float(fila[columna])
                count += 1
            except:
                pass
        if count > 0:
            promedios[columna] = suma / count
    return json.dumps(promedios)

Diagrama: Proceso de decisión en auditoría

🔍 Revisar código
→
¿Problemas críticos?
❌ Sí → Rechazar
✅ No → ¿Problemas mayores?
⚠️ Sí → Requiere cambios
📦 Solo menores → Aprobar

Tareas

  1. Identifica al menos 3 problemas (crítico o menor)
  2. Propón correcciones
  3. Escribe las pruebas mínimas
  4. Decide: ¿Apruebas, requieres cambios o rechazas?

Problemas a identificar: Error si CSV vacío, no maneja archivos inexistentes, mezcla columnas no numéricas.

Consejo del instructor: En una auditoría real, prioriza los problemas críticos sobre los menores. Si el código tiene un solo problema crítico (como fallar con CSV vacío), ya merece "requiere cambios". No apruebes código con problemas bloqueantes, incluso si "funciona" en el caso feliz.

Errores frecuentes en este ejercicio

  • Identificar solo los problemas obvios: El CSV vacío y el archivo inexistente son fáciles de ver; el problema de mezclar columnas requiere más análisis
  • Proponer soluciones sin considerar rendimiento: Cargar el CSV completo en memoria está bien para archivos pequeños, pero no escala
  • No escribir pruebas antes de corregir: Las pruebas deben documentar el comportamiento esperado antes de modificar el código
  • Aprobar el código porque "funciona en mi máquina": El caso feliz no es suficiente; el código debe manejar todos los casos borde

Buenas prácticas para auditar

  • Seguir un proceso sistemático: Usa las 4 pasadas (estructural, lógica, seguridad, mantenibilidad) como guía
  • Documentar cada hallazgo con severidad: Etiqueta cada problema como crítico, mayor o menor para priorizar
  • Escribir pruebas antes de corregir: Las pruebas demuestran el bug y verifican la corrección
  • No corregir sin entender la causa raíz: Pregunta a la IA por qué generó ese código; a veces revela suposiciones incorrectas
  • Tomar una decisión clara: Aprueba solo si no hay problemas críticos o mayores; requiere cambios si hay problemas solucionables; rechaza si la solución es fundamentalmente incorrecta

Actividad práctica

Completa las tareas del enunciado y presenta los resultados en el siguiente formato:

  1. Problemas identificados: Lista numerada con severidad
  2. Código corregido: Versión mejorada de la función
  3. Pruebas escritas: Código de pruebas (mínimo 4 pruebas)
  4. Decisión: Apruebas / Requiere cambios / Rechazas con justificación
Solución de referencia: La función corregida debe validar la existencia del archivo, manejar CSV vacío, detectar columnas numéricas automáticamente, usar processamiento por filas (no cargar todo en memoria), y separar la lógica en funciones más pequeñas. Las pruebas deben cubrir: archivo válido, archivo vacío, archivo inexistente, columnas mixtas, encoding no UTF-8.

Resumen

  • Una auditoría completa identifica problemas de severidad variada (crítico, mayor, menor)
  • Los problemas críticos (CSV vacío, archivo inexistente) son bloqueantes y requieren cambios
  • Las pruebas mínimas deben cubrir cada caso borde identificado
  • La decisión final debe basarse en la severidad de los problemas encontrados, no en el caso feliz
  • Documentar la auditoría permite aprender y mejorar el proceso de revisión
