4.6 Corregir una mala salida de IA

Módulo 4 · Ejercicio

Objetivo: Practicar la identificación y corrección de errores en código generado por IA.

Enunciado

Requerimiento: Función que recibe una lista de números y devuelve la suma de los pares.

Prompt usado: "Haz una función que sume números pares de una lista"

Respuesta de IA:

def suma_pares(lista):
    suma = 0
    for num in lista:
        if num % 2 == 0:
            suma += num
    return suma

Tareas

  1. ¿El código cumple el requerimiento? Identifica problemas
  2. ¿Qué casos borde no maneja?
  3. ¿Qué corregirías o mejorarías?
  4. ¿Qué pruebas escribirías?

Problemas a identificar

  • No maneja lista vacía
  • No maneja tipos no numéricos
  • No maneja números negativos

Solución mejorada

def suma_pares(numeros):
    if not isinstance(numeros, list):
        raise TypeError("Se esperaba una lista")
    return sum(num for num in numeros
               if isinstance(num, (int, float)) and num % 2 == 0)

Introducción

En esta lección pondrás en práctica todo lo aprendido sobre revisión de código generado por IA. Te enfrentarás a un escenario real donde la IA produce una respuesta incorrecta o incompleta, y deberás identificar los problemas, proponer correcciones y justificar tus decisiones. Esta habilidad es fundamental para trabajar con IA de manera profesional.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Identificar errores en código generado por IA de manera sistemática.
  • Aplicar la checklist de revisión para detectar casos borde no manejados.
  • Proponer mejoras concretas a código generado.
  • Escribir pruebas que validen el comportamiento del código en casos borde.
  • Documentar el proceso de corrección para futuras referencias.

Desarrollo del contenido

Análisis del código generado

El código que la IA generó funciona para el caso básico: una lista de números positivos. Sin embargo, un ingeniero profesional debe considerar todos los casos posibles. El error no está en lo que el código hace, sino en lo que no hace.

Casos borde a considerar

  • Lista vacía: Debe devolver 0, no lanzar un error.
  • Tipos no numéricos: Strings, None, booleanos deben manejarse o rechazarse explícitamente.
  • Números negativos pares: -4 es par y debe sumarse.
  • Lista con un solo elemento: Debe funcionar correctamente.
  • Entrada que no es lista: Debe lanzar un error claro.

El proceso de corrección

  1. Identificar: Revisa el código contra la checklist de 8 puntos.
  2. Priorizar: Clasifica los hallazgos en críticos (fallan en producción), mejorables (calidad de código) y aceptables.
  3. Corregir: Implementa las correcciones necesarias.
  4. Verificar: Prueba con casos borde para confirmar que la corrección funciona.
  5. Documentar: Registra qué encontraste y cómo lo solucionaste.

Ejemplos adicionales de correcciones

Ejemplo: Función de búsqueda

Prompt: "Función que busque un usuario por email en una lista"

Código generado (con errores):

def buscar_usuario(email, usuarios):
    for usuario in usuarios:
        if usuario['email'] == email:
            return usuario
    return None

Problemas: No valida tipos, asume que todos los usuarios tienen campo 'email', no maneja email en mayúsculas.

Versión mejorada:

def buscar_usuario(email, usuarios):
    if not isinstance(email, str):
        raise TypeError("El email debe ser un string")
    if not isinstance(usuarios, list):
        raise TypeError("usuarios debe ser una lista")
    email = email.lower().strip()
    for usuario in usuarios:
        if isinstance(usuario, dict) and usuario.get('email', '').lower().strip() == email:
            return usuario
    return None

Ejemplo: Procesamiento de archivos

Prompt: "Función que lea un archivo CSV y devuelva los datos como lista de diccionarios"

Código generado (con errores):

import csv
def leer_csv(archivo):
    with open(archivo) as f:
        return list(csv.DictReader(f))

Problemas: No maneja archivos inexistentes, no especifica encoding, no maneja archivos vacíos.

💡 Consejo del instructor: Cuando corrijas código de IA, no te limites a hacerlo funcionar. Aprovecha para mejorar la calidad general: nombres de variables, estructura, documentación. La IA te da un borrador; tú eres responsable del producto final.
⚠ Errores comunes:
  • Corregir solo el error reportado: Ignorar otros problemas potenciales en el mismo código.
  • No probar la corrección: Asumir que porque la IA corrigió, ahora funciona correctamente.
  • Sobre-corregir: Añadir complejidad innecesaria "por si acaso" sin entender realmente el problema.

Buenas prácticas

  • ✔ Siempre prueba la corrección con casos borde antes de integrarla.
  • ✔ Documenta cada corrección y su justificación.
  • ✔ Si el código original tiene múltiples problemas, abórdalos de uno en uno.
  • ✔ Usa la checklist de revisión para asegurarte de no pasar nada por alto.
  • ✔ Mantén un registro de los errores recurrentes para ajustar tus prompts futuros.

Resumen

  • El código generado por IA puede funcionar para el caso básico pero fallar en casos borde.
  • La corrección efectiva implica identificar, priorizar, corregir, verificar y documentar.
  • Los casos borde más comunes son: valores vacíos, tipos incorrectos y entradas límite.
  • Siempre prueba las correcciones antes de integrarlas.
  • Documentar el proceso de corrección ayuda a mejorar prompts futuros.
