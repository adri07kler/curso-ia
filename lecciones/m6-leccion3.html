6.3 Crear el contexto para la IA

Módulo 6 · Ejercicio

Objetivo: Construir el context packet que se usará con la IA.

Introducción

Una IA sin contexto es como un desarrollador que llega a un proyecto nuevo sin leer la documentación: puede escribir código, pero será genérico, incorrecto o irrelevante. El context packet es el documento que le entrega a la IA toda la información necesaria para generar código alineado con tu proyecto.

En esta lección construirás el context packet para el proyecto final del curso, usando la plantilla de la Lección 3.4. Un buen contexto es la diferencia entre código que "funciona" y código que "resuelve el problema correcto".

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Definir el objetivo y rol que debe asumir la IA en un proyecto
  • Describir el contexto del proyecto de manera estructurada y completa
  • Establecer restricciones técnicas claras para guiar las decisiones de la IA
  • Definir criterios de aceptación que la IA pueda usar como guía
  • Especificar límites explícitos sobre lo que la IA no debe hacer

Desarrollo del contenido

¿Qué es un context packet?

Un context packet es un documento estructurado que se entrega a la IA junto con cada prompt para garantizar que el código generado sea consistente con el proyecto. No es un prompt en sí mismo, sino el "manual de bienvenida" del proyecto. Cada prompt que escribas hará referencia a este contexto.

Componentes del context packet

1. Objetivo y rol de la IA

Define qué esperas de la IA y cómo debe comportarse. Ejemplo para nuestro proyecto:

Rol: Asistente de desarrollo senior especializado en aplicaciones web con JavaScript, HTML/CSS y localStorage.
Objetivo: Generar código funcional y bien estructurado para una aplicación web de gestión de tareas personales, siguiendo las restricciones técnicas y criterios de aceptación definidos.

2. Contexto del proyecto

Describe el proyecto en detalle. Incluye el propósito, los usuarios, las funcionalidades principales y la arquitectura general.

Nombre: TaskFlow — Gestor de tareas personales
Propósito: Aplicación web que permite a usuarios organizar tareas en proyectos, asignar etiquetas, establecer prioridades y fechas de vencimiento, con recordatorios y dashboard de estadísticas.
Usuarios: Usuarios individuales (sin colaboración multi-usuario)
Stack técnico: HTML5, CSS3, JavaScript vanilla, localStorage para persistencia

3. Restricciones técnicas

Las restricciones le indican a la IA qué no puede usar, qué debe priorizar y cómo debe estructurar el código.

  • Sin frameworks externos (React, Vue, Angular)
  • Sin librerías de terceros (no npm, no CDN externo excepto para iconos)
  • Persistencia únicamente con localStorage
  • Diseño responsive (mobile-first)
  • Código en español (nombres de variables, funciones, comentarios)
  • Accesibilidad básica (etiquetas semánticas, ARIA donde aplique)

4. Criterios de aceptación

Son la guía que la IA usará para saber si su código es correcto. Deben ser específicos y verificables.

✅ El usuario puede crear, editar y eliminar proyectos
✅ El usuario puede crear tareas dentro de un proyecto con título, prioridad y fecha
✅ Las tareas se ordenan por prioridad (alta primero) y luego por fecha de vencimiento
✅ El dashboard muestra: total de tareas, tareas completadas, tareas por prioridad
✅ Los recordatorios se muestran como notificaciones en el navegador

5. Qué no debe hacer la IA

Los límites son tan importantes como las instrucciones. Indican a la IA dónde detenerse.

🔴 La IA NO debe:
- Implementar autenticación de usuarios (está fuera de alcance)
- Usar base de datos externa o APIs de terceros
- Generar código de backend (es 100% frontend)
- Agregar funcionalidades no especificadas en los requerimientos
- Optimizar prematuramente (rendimiento antes que funcionalidad)

Cada sección del context packet cumple una función específica. Juntas forman una guía completa para la IA.

Ejemplo de context packet en acción

Sin context packet, si pides "crea una función para guardar tareas", la IA podría generar:

// Sin contexto — la IA asume MongoDB, async/await y npm
const mongoose = require('mongoose');
async function saveTask(taskData) {
  const task = new Task(taskData);
  return await task.save();
}

Con context packet (JavaScript vanilla + localStorage), la misma petición genera:

// Con contexto — alineado al stack del proyecto
function guardarTarea(tarea) {
  const tareas = JSON.parse(localStorage.getItem('tareas') || '[]');
  tareas.push(tarea);
  localStorage.setItem('tareas', JSON.stringify(tareas));
}
💡 Consejo del instructor: El context packet es el documento más valioso de tu flujo de trabajo con IA. Invierte tiempo en hacerlo completo y preciso. Un context packet bien escrito puede reducir las iteraciones de 10 a 2 o 3. Cuando veas que la IA se desvía, no arregles el prompt — arregla el context packet.

Errores frecuentes

  • Contexto demasiado corto: "Haz una app de tareas" no es un context packet
  • Olvidar restricciones técnicas: La IA usará React si no le dices que no
  • No definir el "qué no hacer": La IA agregará funcionalidades que no pediste
  • Contexto contradictorio: Decir "sin frameworks" y luego "usar componentes reutilizables" sin aclarar
  • No actualizar el contexto: El context packet debe evolucionar con el proyecto

Buenas prácticas

  • Escribir el contexto en un archivo separado: Para reutilizarlo en múltiples prompts
  • Ser específico en las restricciones: "localStorage" no "persistencia"
  • Incluir ejemplos de código esperado: La IA entiende mejor con ejemplos
  • Revisar el contexto periódicamente: Lo que no funciona se ajusta
  • Compartir el contexto con el equipo: Todos usan la misma guía para prompts consistentes

Actividad práctica

  1. Define el objetivo y rol esperado de la IA
  2. Describe el contexto del proyecto
  3. Establece restricciones técnicas
  4. Define criterios de aceptación
  5. Especifica qué no debe hacer la IA

Mini reto

Pide a una IA que genere un context packet para el proyecto de la idea que trabajaste en lecciones anteriores. Luego evalúa el resultado: ¿qué secciones incluyó? ¿Qué información crucial omitió? Compara el context packet generado por la IA con el que tú escribiste y documenta las diferencias. ¿Cuál es más completo y por qué?

Resumen

  • El context packet es el documento que guía a la IA durante todo el proyecto
  • Debe incluir: rol, contexto, restricciones técnicas, criterios de aceptación y límites
  • Un buen contexto reduce drásticamente las iteraciones necesarias
  • Las restricciones técnicas evitan que la IA use tecnologías no autorizadas
  • Los límites ("qué no hacer") son tan importantes como las instrucciones positivas
  • El context packet debe actualizarse a medida que el proyecto evoluciona
