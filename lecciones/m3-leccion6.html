3.6 Crear una cadena de prompts

Módulo 3 · Ejercicio

Objetivo: Practicar la creación de cadenas de prompts para un problema real.

Introducción

En la lección anterior aprendiste la teoría de la encadenación de prompts. Ahora es momento de aplicarlo. Este ejercicio te guiará paso a paso para que construyas tu propia cadena de prompts a partir de un requerimiento realista del mundo del desarrollo de software.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Aplicar la cadena mínima de 6 pasos a un problema real de desarrollo.
  • Redactar prompts específicos y efectivos para cada paso de la cadena.
  • Construir un Context Packet coherente para el problema planteado.
  • Evaluar críticamente la calidad de su propia cadena de prompts.

Enunciado

Dado el requerimiento:

"Dashboard para usuarios que muestra sus últimas 10 transacciones, su saldo actual, y un gráfico de gastos por categoría del mes actual. Solo datos de los últimos 30 días. Debe actualizarse cada 5 minutos."

Crea una cadena de 6 prompts:

  1. Prompt 1: Analiza el requerimiento y detecta ambigüedades
  2. Prompt 2: Convierte en especificación clara
  3. Prompt 3: Propón plan de implementación por pasos
  4. Prompt 4: Implementa solo el primer paso
  5. Prompt 5: Genera pruebas para validar
  6. Prompt 6: Revisa críticamente el resultado

Incluye también el context packet necesario.

Guía paso a paso

Paso 1: Context Packet

Antes de empezar la cadena, define el Context Packet del proyecto. Esto asegura que la IA tenga toda la información de fondo necesaria.

## Objetivo:
Dashboard financiero para usuarios de una app de banca personal

## Rol:
Desarrollador frontend senior especializado en dashboards

## Contexto:
App web React 18 con TypeScript, datos simulados (mock API)

## Estado actual:
Proyecto Vite + React 18 + TypeScript + Tailwind CSS
Estructura: /components, /hooks, /services, /types
Mock API en /services/api.ts con datos de prueba

## Restricciones:
- React 18, TypeScript strict mode
- Recharts para gráficos
- Tailwind CSS
- Sin backend real (datos mock)
- Actualización cada 5 minutos (setInterval)

## Criterios de aceptación:
- Muestra últimas 10 transacciones
- Muestra saldo actual
- Gráfico de gastos por categoría del mes actual
- Datos filtrados a últimos 30 días
- Actualización automática cada 5 minutos
- Diseño responsive

## Formato:
Código completo de cada componente

## No hacer:
- No modificar configuraciones existentes
- No agregar dependencias nuevas
- No implementar backend real

## Revisión humana:
- Verificar que los datos mostrados sean correctos
- Validar que el gráfico se renderice adecuadamente

Paso 2: Prompt 1 - Analizar ambigüedades

El primer prompt debe detectar qué información falta o es ambigua en el requerimiento.

Con el siguiente Context Packet, analiza el requerimiento del dashboard financiero y detecta TODAS las ambigüedades que encuentres.

Requerimiento: "Dashboard para usuarios que muestra sus últimas 10 transacciones, su saldo actual, y un gráfico de gastos por categoría del mes actual. Solo datos de los últimos 30 días. Debe actualizarse cada 5 minutos."

Para cada ambigüedad, explica por qué es relevante y qué información adicional necesitamos.

Paso 3: Prompt 2 - Especificación clara

Basándote en el análisis anterior, escribe una especificación técnica clara y completa del dashboard financiero. La especificación debe incluir: componentes, datos, interacciones y reglas de negocio.

Paso 4: Prompt 3 - Plan de implementación

Propón un plan de implementación detallado paso a paso. Cada paso debe incluir: archivos a modificar/crear, componentes involucrados y criterios para considerar el paso completado.

Paso 5: Prompt 4 - Implementar paso 1

Implementa solo el primer paso del plan de implementación. Debes generar el código completo siguiendo las restricciones del Context Packet.

Paso 6: Prompt 5 - Generar pruebas

Genera pruebas unitarias con Jest para validar el código del paso 1. Las pruebas deben cubrir: casos normales, casos borde y errores esperados.

Paso 7: Prompt 6 - Revisión crítica

Revisa críticamente el código generado en el paso 1. Identifica: problemas de rendimiento, violaciones de las restricciones del proyecto, oportunidades de mejora en legibilidad y mantenibilidad. Sé específico: menciona archivos y líneas.

Cada prompt (P1-P6) recibe el resultado del anterior y construye sobre él
Consejo del instructor: No copies y pegues estos prompts sin entenderlos. Personaliza cada prompt según el problema específico. La calidad de la cadena depende de la precisión de cada eslabón.

Errores frecuentes

  • Prompts demasiado genéricos: "Analiza esto" sin indicar el formato de respuesta esperado. Sé específico: "Analiza y lista en formato de viñetas".
  • Saltar la revisión entre pasos: Si no revisas el resultado de cada paso, los errores se acumulan. Detente después de cada respuesta de la IA y evalúa.
  • No proporcionar el Context Packet: Sin contexto, la IA asumirá tecnologías y decisiones que pueden no coincidir con tu proyecto.

Buenas prácticas

  • Revisa el resultado de cada prompt antes de enviar el siguiente.
  • Si un resultado no es satisfactorio, ajusta el prompt y vuelve a intentarlo.
  • Documenta los 6 prompts y sus resultados como referencia futura.
  • Comparte la cadena completa con tu equipo para estandarizar procesos.
  • Mide el tiempo total vs la calidad obtenida para iterar y mejorar.

Actividad práctica

Completa los 6 prompts de la cadena para el dashboard financiero. No es necesario que ejecutes los prompts con una IA (aunque puedes hacerlo si lo deseas), pero debes redactar cada prompt con el nivel de detalle necesario para obtener una respuesta de calidad.

Adicionalmente, incluye el Context Packet que hayas definido para el proyecto.

Mini reto

Sin usar el Context Packet proporcionado en esta lección, crea tu propio Context Packet para el dashboard financiero desde cero. Luego compara tu versión con la de la lección. ¿Qué agregaste? ¿Qué omitiste? ¿Cuál versión crees que produciría mejores resultados con la IA?

Resumen

  • La cadena de 6 prompts transforma un requerimiento vago en un resultado de alta calidad.
  • El Context Packet es el cimiento de toda la cadena.
  • Cada prompt debe ser específico, con formato de respuesta definido.
  • La revisión entre pasos es obligatoria, no opcional.
  • Documentar la cadena completa permite reutilizarla en proyectos similares.
