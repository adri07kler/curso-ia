4.1 Qué es el desarrollo asistido por IA

Módulo 4 · Video

Objetivo: Definir el concepto de desarrollo asistido y en qué se diferencia de la generación automática de código.

Desarrollo asistido ≠ Código automático

La IA asiste, no reemplaza. El ingeniero mantiene el control en cada paso.

El flujo correcto

  1. El ingeniero define qué necesita
  2. La IA propone una solución
  3. El ingeniero revisa críticamente
  4. El ingeniero prueba la solución
  5. El ingeniero decide aceptar, modificar o rechazar

Herramientas

Chat-based (ChatGPT, Claude), integradas en IDE (Copilot, Codeium), o APIs.

Lo que cambia

El tiempo se redistribuye: menos tiempo escribiendo código, más tiempo diseñando, revisando y probando.

Introducción

El desarrollo asistido por IA está transformando la manera en que los ingenieros de software construyen aplicaciones. En esta lección explorarás cómo integrar la IA como un compañero de desarrollo que acelera tu productividad sin reemplazar tu criterio técnico. Aprenderás a diferenciar entre una herramienta que asiste y una que automatiza ciegamente, un concepto clave para mantener la calidad en tus proyectos.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Comprender qué es el desarrollo asistido por IA y cómo se diferencia de la generación automática de código.
  • Identificar el flujo correcto de trabajo al usar IA como asistente de desarrollo.
  • Reconocer las herramientas disponibles (ChatGPT, Claude, Copilot, Codeium) y su aplicación según el contexto.
  • Analizar cómo se redistribuye el tiempo del desarrollador al incorporar IA en su flujo de trabajo.
  • Aplicar el criterio humano como filtro indispensable antes de aceptar código generado.

Desarrollo del contenido

¿Qué es exactamente el desarrollo asistido por IA?

El desarrollo asistido por IA es un enfoque donde el programador utiliza modelos de lenguaje (LLMs) como herramientas de apoyo para escribir, revisar, optimizar y documentar código. A diferencia de la generación automática, donde se espera que la IA produzca un sistema completo sin supervisión, el desarrollo asistido mantiene al ingeniero en el centro de cada decisión.

El rol del ingeniero en el nuevo flujo

En un flujo tradicional, el desarrollador escribe cada línea desde cero. Con asistencia de IA, el rol evoluciona: el ingeniero se convierte en un arquitecto que define qué construir, revisa lo generado, y toma decisiones críticas sobre qué aceptar, modificar o rechazar. Este cambio exige fortalecer habilidades de revisión de código, pensamiento crítico y diseño de sistemas.

Herramientas según el contexto

No todas las herramientas de IA funcionan igual. Los asistentes chat-based como ChatGPT y Claude son ideales para explorar soluciones, depurar errores y obtener explicaciones. Los integrados en IDE como Copilot y Codeium destacan en autocompletado en tiempo real y sugerencias contextuales. Las APIs (OpenAI, Anthropic) permiten automatizar flujos completos. La clave es elegir la herramienta adecuada para cada tarea.

Ejemplos

Ejemplo 1: Desarrollo web con asistencia de IA

Escenario: Necesitas crear un componente de tarjeta de producto para un e-commerce.

Sin IA: Escribes manualmente el HTML, CSS y JS, investigando en documentación cada propiedad.

Con IA: Le pides a ChatGPT que genere el componente con especificaciones concretas: "Crea una tarjeta de producto con imagen, título, precio y botón de compra, usando flexbox y diseño responsivo". Revisas el código, ajustas estilos y lo integras.

Ejemplo 2: Automatización de scripts

Escenario: Debes procesar 10,000 archivos CSV para extraer datos específicos.

Sin IA: Escribes un script manualmente, probando cada función una por una.

Con IA: Pides: "Genera un script en Python que lea una carpeta de CSVs, extraiga las columnas 'nombre' y 'total', y las consolide en un solo archivo Excel". Revisas el código generado, ajustas rutas, y ejecutas.

Ejemplo 3: Prompt Engineering para refactorización

Escenario: Tienes una función legacy de 200 líneas que necesita ser refactorizada.

Con IA: Proporcionas el código original y pides: "Refactoriza esta función dividiéndola en funciones más pequeñas con nombres descriptivos. Cada función debe tener una única responsabilidad. Mantén la misma funcionalidad".

💡 Consejo del instructor: La IA es excelente generando código, pero no entiende el contexto completo de tu proyecto. Siempre mantén un archivo de contexto compartido donde documentes las decisiones clave de arquitectura. Esto te ayudará a ti y a la IA a mantener coherencia.
⚠ Errores comunes:
  • Delegar sin supervisión: Asumir que el código de la IA es correcto sin revisarlo.
  • Prompts vagos: Pedir "haz un CRUD" sin especificar tecnología, estructura o restricciones.
  • No versionar: No usar git porque "la IA lo puede regenerar". Siemén versiona tu código.

Buenas prácticas

  • ✔ Revisa siempre el código generado antes de integrarlo.
  • ✔ Divide problemas grandes en prompts pequeños y específicos.
  • ✔ Mantén un contexto compartido con la IA (archivo de especificaciones).
  • ✔ Prueba el código generado en un entorno aislado antes de producción.
  • ✔ Documenta las decisiones importantes para mantener coherencia.
  • ✔ Usa control de versiones en cada paso del desarrollo asistido.

Actividad práctica

Escenario: Tienes que implementar un validador de formularios en JavaScript que verifique email, teléfono y contraseña.

  1. Escribe un prompt para solicitar a la IA la función validadora. Sé específico sobre: formato de email, longitud mínima de contraseña, y formato de teléfono.
  2. Ejecuta el prompt en ChatGPT, Claude o similar.
  3. Revisa críticamente el código generado usando la checklist de la lección 4.5.
  4. Identifica al menos 2 mejoras o correcciones necesarias.
  5. Documenta el proceso: prompt usado, código recibido, correcciones aplicadas.

Mini reto

Implementa un generador de prompts reutilizable. Crea una función que, dados: tecnología (React, Vue, Python), tipo de componente (formulario, tabla, API) y requerimientos específicos, genere automáticamente un prompt estructurado y listo para usar con la IA. El prompt debe incluir contexto, restricciones y formato de salida esperado.

Resumen

  • El desarrollo asistido por IA no reemplaza al ingeniero, lo potencia.
  • El flujo correcto: definir → solicitar → revisar → probar → decidir.
  • Las herramientas se eligen según el contexto (chat, IDE, API).
  • El tiempo del desarrollador se redistribuye hacia diseño, revisión y pruebas.
  • La calidad del resultado depende del criterio humano, no de la IA.
