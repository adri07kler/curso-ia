3.5 Encadenación de prompts

Módulo 3 · Video

Objetivo: Aprender a dividir un problema grande en una secuencia de prompts pequeños y manejables.

Introducción

Cuando enfrentas un problema complejo, pedirle a la IA que lo resuelva todo en un solo prompt produce resultados genéricos, inconsistentes o directamente incorrectos. La técnica de encadenación de prompts divide el problema en pasos secuenciales, donde cada prompt se beneficia del resultado del anterior.

En esta lección aprenderás a diseñar cadenas de prompts que produzcan resultados de alta calidad, paso a paso.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Identificar cuándo un problema requiere múltiples prompts en lugar de uno solo.
  • Diseñar una cadena de prompts secuenciales donde cada paso dependa del anterior.
  • Explicar los beneficios de la encadenación frente al mega-prompt.
  • Aplicar la cadena mínima de 6 pasos a problemas reales de desarrollo.

Problema del mega-prompt

Un prompt enorme que pide todo a la vez produce respuestas genéricas, inconsistentes o con errores.

¿Por qué falla el mega-prompt?

  • La IA pierde el foco cuando hay demasiadas instrucciones simultáneas.
  • No puede verificar la coherencia entre todas las partes del resultado.
  • Si una parte es incorrecta, todo el resultado se descarta.
  • Consume muchos tokens en una sola llamada sin posibilidad de corrección intermedia.

Encadenación

Dividir el trabajo en pasos secuenciales donde cada prompt usa el resultado del anterior.

En lugar de pedir "crea un sistema de facturación completo", divides el problema en:

  1. Analizar requerimientos
  2. Diseñar la arquitectura
  3. Implementar el modelo de datos
  4. Implementar la lógica de negocio
  5. Implementar la interfaz de usuario
  6. Generar pruebas

Beneficios

  • Cada prompt es simple y específico
  • Puedes revisar y corregir en cada paso
  • Mayor calidad en el resultado final
  • Menos tokens desperdiciados
  • Puedes reutilizar pasos intermedios en otras tareas
  • El proceso es reproducible y documentable

Cadena mínima de prompts

  1. Analiza el requerimiento y detecta ambigüedades
  2. Convierte el requerimiento en especificación clara
  3. Propón plan de implementación por pasos
  4. Implementa solo el paso 1
  5. Genera pruebas para validar
  6. Revisa críticamente el resultado

Tabla comparativa: Mega-prompt vs Cadena de prompts

Ejemplo real: API de reservas

Paso 1: Analizar requerimiento

Prompt: "Analiza este requerimiento y lista las ambigüedades que encuentres:
'API para reservar mesas en restaurantes. Los clientes reservan, el restaurante confirma.'"

Respuesta esperada: ¿Cuántas mesas? ¿Horarios? ¿Capacidad? ¿Cómo se confirma? ¿Notificaciones?

Paso 2: Especificación clara

Prompt: "Con base en las ambigüedades detectadas, escribe una especificación clara del sistema de reservas."

Paso 3: Plan de implementación

Prompt: "Propón un plan de implementación por pasos, empezando por el modelo de datos."

Paso 4: Implementar paso 1

Prompt: "Implementa solo el modelo de datos (Prisma schema) según la especificación."

Paso 5: Pruebas

Prompt: "Genera pruebas unitarias para el modelo de datos usando Jest."

Paso 6: Revisión crítica

Prompt: "Revisa críticamente el modelo de datos. Identifica problemas de normalización, rendimiento y escalabilidad."
Consejo del instructor: No intentes saltarte pasos. Cada paso de la cadena tiene un propósito: detectar errores temprano, refinar la dirección y asegurar calidad. Saltar un paso es como construir una casa sin cimientos.

Errores frecuentes

  • Hacer pasos demasiado grandes: Si un paso aún requiere múltiples decisiones, divídelo en pasos más pequeños.
  • No verificar el resultado intermedio: Cada paso produce un resultado que debes revisar antes de continuar. Si el paso 1 tiene errores, toda la cadena se contamina.
  • Perder el contexto entre pasos: Asegúrate de que la IA recuerde el contexto del proyecto en cada paso de la cadena.

Buenas prácticas

  • Define la cadena completa antes de empezar, no la improvises paso a paso.
  • Revisa y aprueba cada paso antes de enviar el siguiente prompt.
  • Documenta cada paso y su resultado para futuras referencias.
  • Usa la cadena mínima de 6 pasos como punto de partida y adáptala según el problema.
  • Si un paso produce un resultado insatisfactorio, repítelo con instrucciones más precisas antes de avanzar.

Actividad práctica

Toma el siguiente requerimiento incompleto y aplica la cadena mínima de 6 prompts:

"Sistema de gestión de inventario para un almacén. Debe controlar entradas y salidas de productos, generar alertas de stock bajo y reportes semanales."
  1. Escribe el prompt para detectar ambigüedades.
  2. Escribe la especificación clara resultante.
  3. Escribe el plan de implementación.
  4. Escribe el prompt para implementar solo el modelo de datos.
  5. Escribe el prompt para generar pruebas.
  6. Escribe el prompt para revisión crítica.

Mini reto

Selecciona un proyecto open-source que conozcas. Imagina que necesitas agregar una funcionalidad nueva (ej: "modo oscuro" en una app existente). Diseña una cadena de 6 prompts específicos para esa funcionalidad, considerando que ya existe el código base.

Resumen

  • Los mega-prompts fallan porque la IA pierde el foco con instrucciones simultáneas.
  • La encadenación divide el problema en pasos secuenciales y manejables.
  • Cada paso recibe el resultado del anterior, mejorando la calidad progresivamente.
  • La cadena mínima son 6 pasos: analizar → especificar → planificar → implementar → probar → revisar.
  • Revisar cada paso antes de continuar es fundamental para el éxito de la cadena.
