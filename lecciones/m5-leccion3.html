5.3 Casos borde y errores esperados

Módulo 5 · Documento

Objetivo: Aprender a identificar y probar sistemáticamente los casos borde.

Introducción

La IA suele generar código que funciona perfectamente con la entrada "típica" pero falla estrepitosamente cuando recibe datos inesperados. Un usuario que ingresa un número negativo, un archivo vacío, o una conexión de red que se cae — la IA rara vez anticipa estos escenarios. Esta lección te enseñará una estrategia sistemática para encontrar casos borde antes de que tus usuarios los encuentren por ti.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Aplicar una estrategia de 7 pasos para identificar casos borde
  • Escribir código robusto que maneje entradas vacías, inválidas y límites
  • Anticipar fallos en dependencias externas, concurrencia y tiempo
  • Implementar validaciones defensivas en código generado por IA

Estrategia para encontrar casos borde

  1. Entrada vacía/nula: ¿Qué pasa si no hay datos? Ej: lista vacía, string vacío, None, null
  2. Entrada mínima/máxima: ¿Valores en los límites? Ej: 0, 1, -1, MAX_INT, Infinity
  3. Entrada inválida: ¿Tipos incorrectos? Ej: string donde se espera número, objeto donde se espera array
  4. Estado del sistema: ¿BD caída? ¿Red lenta? Ej: timeout, conexión rechazada
  5. Concurrencia: ¿Dos peticiones simultáneas? Ej: race conditions, datos corruptos
  6. Dependencias: ¿API externa devuelve error? Ej: 500, 404, rate limit
  7. Tiempo: ¿Timeout? Ej: operación que excede el tiempo máximo permitido

Aplicación práctica de la estrategia

Cada vez que generes código con IA, recorre mentalmente estos 7 puntos y pregúntate: "¿Qué pasaría aquí?". Si la IA no manejó un escenario, agrega la validación manualmente. Con la práctica, este chequeo toma menos de 2 minutos.

Ejemplo completo

def get_usuario_por_id(db, usuario_id):
    if not db:
        raise ConnectionError("BD no disponible")
    if not isinstance(usuario_id, int):
        raise TypeError("ID debe ser entero")
    if usuario_id <= 0:
        raise ValueError("ID debe ser positivo")
    resultado = db.query("SELECT * FROM usuarios WHERE id = ?", usuario_id)
    return resultado[0] if resultado else None

Análisis de casos borde en el ejemplo

  • Entrada vacía/nula: db=None → raise ConnectionError ✓; usuario_id=None → raise TypeError ✓
  • Entrada mínima/máxima: usuario_id=0 → raise ValueError ✓; usuario_id=-5 → raise ValueError ✓
  • Entrada inválida: usuario_id="abc" → raise TypeError ✓
  • Dependencias: db.query() falla → no hay try/except ⚠️ posible mejora
  • Resultado vacío: resultado=[] → retorna None ✓ (manejado correctamente)

💡 Pregunta clave: ¿Qué pasaría aquí si la entrada fuera inesperada?
Consejo del instructor: Aplica el Principio de Robustez (Postel's Law): "Sé conservador en lo que envías, sé liberal en lo que aceptas". Para código generado por IA, esto significa: valida estrictamente las entradas y maneja gracefully cualquier escenario inesperado.

Errores frecuentes

  • Validar solo el tipo: No basta con verificar que es entero; también hay que verificar rango y valores válidos
  • Ignorar el contexto: Un caso borde en desarrollo web (header malformado) es distinto a uno en data science (NaN en datos)
  • Manejar errores genéricamente: Un except: pass oculta errores que deberían ser visibles
  • No probar los casos borde: Identificarlos no basta; hay que escribir pruebas que los verifiquen

Buenas prácticas

  • Validar al inicio de la función: Antes de cualquier operación, verifica todas las entradas (fail fast)
  • Usar tipos específicos de excepción: Prefiere ValueError, TypeError, ConnectionError sobre Exception genérico
  • Documentar los casos borde: En el docstring de la función, menciona qué entradas pueden fallar
  • Aplicar la estrategia de 7 pasos: Recorre la lista mentalmente cada vez que generes código con IA
  • Escribir pruebas para cada borde: Por cada caso borde identificado, escribe una prueba automatizada

Actividad práctica

La IA ha generado esta función que divide dos números:

def dividir(a, b):
    return a / b

Aplica la estrategia de 7 pasos para identificar los casos borde. Luego:

  1. Escribe una versión mejorada que maneje todos los casos borde
  2. Escribe pruebas para cada caso borde identificado
  3. ¿Qué pasa con b = 0? ¿Y si a o b son None? ¿Y si son strings numéricos?

Mini reto

Genera con IA una función que lea un archivo JSON y devuelva la suma de los valores numéricos. Sin modificar el código generado, aplica la estrategia de 7 pasos para encontrar todos los casos borde. Documenta cada caso borde y escribe las pruebas correspondientes. ¿Cuántos casos encontraste? Comparte tu lista con un compañero y comparen resultados.

Resumen

  • La estrategia de 7 pasos cubre: entrada vacía, valores límite, tipos inválidos, estado del sistema, concurrencia, dependencias y tiempo
  • El código generado por IA típicamente no maneja casos borde; es responsabilidad del desarrollador agregarlos
  • Validar al inicio (fail fast) evita errores complejos más adelante
  • Usar excepciones específicas mejora la legibilidad y el debugging
  • Cada caso borde identificado debe tener una prueba automatizada
