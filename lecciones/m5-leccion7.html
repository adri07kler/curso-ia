5.7 Quiz de validación

Módulo 5 · Quiz

Objetivo: Evaluar conocimientos sobre validación, pruebas y documentación.

Introducción

Este quiz pondrá a prueba los conceptos fundamentales que has aprendido en el Módulo 5: los riesgos de confiar ciegamente en la IA, las pruebas mínimas necesarias, la identificación de casos borde, las técnicas de code review para código generado, y la documentación de prompts y decisiones. Cada pregunta está diseñada para evaluar comprensión, no memorización — piensa cuidadosamente antes de responder.

Objetivos de aprendizaje

Al finalizar este quiz el estudiante será capaz de:

  • Demostrar comprensión de los riesgos y limitaciones del código generado por IA
  • Aplicar criterios de validación y prueba a situaciones reales de desarrollo
  • Evaluar decisiones sobre cuándo y cómo usar IA en proyectos de software

Errores frecuentes

  • Confundir "prueba de humo" con "prueba de seguridad": La prueba de humo verifica ejecución básica; la seguridad se revisa por separado
  • Pensar que la IA requiere menos revisión: Es lo opuesto — la IA comete errores poco convencionales que requieren más atención
  • Aceptar código que no se entiende: Aunque pase pruebas, si no entiendes el código, no puedes mantenerlo ni auditarlo
  • Subestimar los casos borde: Un código que falla con entrada vacía no está listo para producción, incluso si funciona con datos válidos

Buenas prácticas

  • Siempre escribir pruebas mínimas: Caso feliz + caso borde por cada función generada por IA
  • Revisar código de IA con más atención que código humano: La IA comete errores inesperados
  • Documentar prompts, decisiones y pruebas: Para trazabilidad y mejora continua
  • Nunca integrar código sin entenderlo: Si no lo entiendes, pide explicación o no lo uses
  • Aplicar el criterio de auditoría: Problemas críticos = requiere cambios, no aprobar

Resumen

  • La prueba mínima indispensable es la de humo (caso feliz)
  • El código generado por IA requiere más revisión que el código humano, no menos
  • Nunca aceptes código que no entiendes completamente
  • Documentar el uso de IA es esencial para trazabilidad y aprendizaje del equipo
  • Un caso borde no manejado es razón suficiente para no aprobar código en producción
  • La responsabilidad final del código siempre es del desarrollador, no de la IA
