3.4 Plantilla de contexto

Módulo 3 · Documento

Objetivo: Proporcionar una plantilla reutilizable para estructurar el contexto.

Introducción

Hasta ahora has aprendido qué información incluir y qué eliminar. El siguiente paso es estandarizar ese conocimiento en una plantilla reutilizable: el Context Packet. Esta plantilla te permitirá iniciar cualquier sesión con IA de manera estructurada y profesional, asegurando que nunca olvides ningún elemento esencial.

En esta lección aprenderás a usar y personalizar el Context Packet para cualquier proyecto de desarrollo.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Utilizar la plantilla Context Packet para estructurar información para la IA.
  • Personalizar la plantilla según el tipo de proyecto (web, automatización, análisis de datos, etc.).
  • Mantener y versionar el Context Packet como parte de la documentación del proyecto.
  • Compartir el Context Packet con el equipo para estandarizar la interacción con IA.

Context Packet

El Context Packet es un documento estructurado que contiene toda la información que la IA necesita para entender tu proyecto. Piénsalo como el README de tu proyecto, pero orientado a la IA.

## Objetivo:
[¿Qué vamos a construir/lograr?]

## Rol esperado de la IA:
[¿Cómo debe comportarse?]

## Contexto del proyecto:
[Descripción breve del proyecto y estado actual]

## Estado actual:
[¿Qué código ya existe?]

## Restricciones técnicas:
- Lenguaje/framework:
- BD:
- Dependencias permitidas:
- Convenciones:

## Criterios de aceptación:
- [Criterio 1]
- [Criterio 2]

## Formato de salida esperado:
[ej: "Código completo", "Pseudocódigo"]

## Qué no debe hacer la IA:
- [No modificar archivos X]
- [No agregar dependencias]

## Punto de revisión humana:
[¿Qué debe revisar el humano?]

Cómo usar el Context Packet

Paso 1: Crear el archivo

Crea un archivo llamado context-packet.md en la raíz de tu proyecto y complétalo con la información de tu proyecto.

Paso 2: Iniciar la sesión

Al iniciar una sesión con la IA, pega el contenido completo del Context Packet como primer mensaje.

Paso 3: Refrescar cuando sea necesario

Cuando cambien las condiciones del proyecto (nuevas dependencias, cambios de versión, etc.), actualiza el Context Packet.

Plantillas por tipo de proyecto

Plantilla para proyecto web

## Objetivo:
App web para [descripción del negocio]

## Rol:
Desarrollador full-stack senior

## Contexto:
Proyecto en desarrollo activo, rama main en producción

## Estado actual:
Frontend: [framework] con [librerías]
Backend: [lenguaje/framework] con [BD]
Autenticación: [sistema]
Despliegue: [plataforma]

## Restricciones:
- Versiones: [especificar]
- Testing: [framework]
- CI/CD: [plataforma]

## Criterios de aceptación:
- Pasa tests existentes
- Sigue guías de estilo del proyecto
- Sin breaking changes

## Formato:
Código completo + tests

## No hacer:
- No tocar configuraciones de infraestructura
- No modificar variables de entorno

## Revisión humana:
- Validar seguridad
- Validar rendimiento

Plantilla para automatización/scripts

## Objetivo:
Script para [tarea específica]

## Rol:
Desarrollador de automatización Python

## Contexto:
Scripts existentes en /scripts, entrada estándar y salida a consola/archivo

## Estado actual:
[scripts existentes]

## Restricciones:
- Python 3.11+ estándar
- Sin dependencias externas no listadas
- Manejo de errores obligatorio
- Logging básico

## Criterios:
- Ejecución sin errores
- Tiempo de ejecución menor a [X] segundos
- Log claro de cada paso

## Formato:
Script Python autocontenido

## No hacer:
- No borrar archivos de entrada
- No sobrescribir sin confirmación

## Revisión:
- Probar con datos reales
- Validar edge cases

Ejemplo completo: Dashboard financiero

## Objetivo:
Dashboard financiero personal que muestra gastos, ingresos y proyecciones mensuales

## Rol:
Desarrollador frontend senior especializado en visualización de datos

## Contexto del proyecto:
App web en React 18 con Vite. Diseño mobile-first. Usuarios individuales (sin auth).

## Estado actual:
- Proyecto creado con Vite + React 18 + TypeScript
- Tailwind CSS configurado
- Estructura de carpetas: /components, /hooks, /utils, /types
- Datos mock en /data/mockTransactions.ts

## Restricciones técnicas:
- React 18, TypeScript strict mode
- Tailwind CSS (no CSS puro ni módulos)
- Recharts para gráficos (ya instalado)
- No usar Redux ni Zustand (preferir useState + useContext)
- Node 20 LTS

## Criterios de aceptación:
- Componentes tipados con TypeScript
- Responsive (mobile y desktop)
- Gráficos responsivos
- Datos mock actualizados al mes actual
- Sin errores de compilación

## Formato de salida:
Código completo de cada componente nuevo o modificado

## Qué no debe hacer:
- No modificar package.json
- No agregar dependencias
- No tocar configuración de Vite o Tailwind

## Punto de revisión humana:
- Validar que los tipos sean correctos
- Verificar que los gráficos se rendericen correctamente
- Revisar accesibilidad básica

Tabla comparativa: Context Packet vs Documentación tradicional

Aspecto Documentación tradicional Context Packet
Propósito Entender el proyecto como humano Preparar el contexto óptimo para IA
Estructura Libre, narrativa, variable Estandarizada con secciones fijas
Contenido típico Historial del proyecto, decisiones de negocio, diagramas UML Stack, rol, restricciones, criterios de aceptación

2. Rol esperado de la IA
3. Estado actual del proyecto
4. Restricciones técnicas
5. Criterios de aceptación
6. Formato de salida esperado
7. Lo que NO debe hacer
8. Punto de revisión humana
El Context Packet contiene 8 secciones que preparan a la IA para generar respuestas precisas
Consejo del instructor: Guarda tu Context Packet en un servicio de notas sincronizado (Notion, Obsidian, etc.) para tenerlo siempre disponible. Si trabajas en equipo, súbelo al repositorio del proyecto para que todos usen la misma versión.

Errores frecuentes

  • Context Packet demasiado genérico: Si tu plantilla sirve para cualquier proyecto, no está haciendo su trabajo. Personaliza cada sección con información específica.
  • No actualizar el Context Packet: Un packet desactualizado es peor que no tenerlo, porque la IA confiará en información incorrecta.
  • Olvidar la sección de revisión humana: La IA no debe tomar decisiones críticas sin supervisión. Especifica claramente qué debe validar un humano.

Buenas prácticas

  • Versiona el Context Packet junto con el código (commitea los cambios).
  • Incluye enlaces a documentación relevante dentro del packet.
  • Usa el mismo packet para todas las sesiones de IA del mismo proyecto.
  • Revisa y actualiza el packet al menos una vez por sprint.
  • Comparte la plantilla con tu equipo para estandarizar la interacción con IA.

Actividad práctica

Crea un Context Packet completo para uno de los siguientes proyectos:

  1. Una API REST para gestión de inventario de una tienda.
  2. Un script de automatización que descargue, procese y envíe por correo reportes diarios.
  3. Un componente React de calendario con eventos arrastrables.

Incluye todas las secciones del Context Packet y asegúrate de que cada campo contenga información específica y útil.

Mini reto

Toma un proyecto tuyo existente y crea su Context Packet. Luego, pídele a la IA que realice una tarea de ese proyecto usando solo el Context Packet como información de fondo. Compara la respuesta con una sesión anterior donde no usaste el packet. ¿Cuál fue mejor?

Resumen

  • El Context Packet estandariza la información que le das a la IA.
  • Debe ser específico para cada proyecto, no genérico.
  • Se versiona junto con el código y se actualiza periódicamente.
  • Incluye siempre la sección de revisión humana para tareas críticas.
  • Usar el mismo packet en todas las sesiones garantiza consistencia.
