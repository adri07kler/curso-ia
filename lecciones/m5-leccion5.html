5.5 Documentar prompts y decisiones

Módulo 5 · Documento

Objetivo: Establecer la práctica de documentar el uso de IA para trazabilidad y mejora continua.

Introducción

Cuando un equipo usa IA para generar código, la falta de documentación crea un problema de trazabilidad: meses después, nadie sabe qué partes fueron generadas por IA, qué prompts se usaron, ni por qué se aceptaron o rechazaron ciertas soluciones. Esto convierte el código en una caja negra. Documentar el uso de IA no es burocracia — es una práctica de ingeniería que permite aprender de los errores, repetir los aciertos y auditar el proceso.

Objetivos de aprendizaje

Al finalizar esta lección el estudiante será capaz de:

  • Comprender por qué documentar el uso de IA es crítico para trazabilidad y mejora continua
  • Crear un registro estructurado de prompts, respuestas y decisiones
  • Identificar patrones de error recurrentes en código generado por IA
  • Aplicar un formato de documentación reusable en proyectos reales

¿Por qué documentar?

  • Trazabilidad: Saber qué partes fueron generadas por IA
  • Reproducibilidad: Poder regenerar si se pierde el código
  • Aprendizaje: Ver qué prompts funcionaron
  • Auditoría: Para equipos que requieren cumplimiento
  • Mejora continua: Identificar patrones de error recurrentes

El costo de no documentar

Sin documentación, cada miembro del equipo genera código con IA de forma aislada. No hay aprendizaje colectivo: el error que cometió un desarrollador lo repetirá otro. Los prompts exitosos se pierden. Los patrones de fallo —como que la IA genera SQL injection consistentemente— nunca se identifican. Documentar transforma el uso individual de IA en conocimiento del equipo.

Qué documentar

  1. El requerimiento
  2. El contexto entregado
  3. Los prompts utilizados
  4. La respuesta generada
  5. Cambios aceptados (y por qué)
  6. Cambios rechazados (y por qué)
  7. Pruebas realizadas
  8. Errores encontrados
  9. Conclusión

Plantilla de registro

## Registro de uso de IA
**Fecha:** 2026-07-01
**Requerimiento:** Función que valida formato de email
**Prompt:** "Escribe una función en Python que valide..."
**Respuesta IA:** [pegar código generado]
**Cambios aceptados:** Se usó la validación de dominio
**Cambios rechazados:** No se usó la expresion regular (demasiado permisiva)
**Pruebas:** Caso feliz, email inválido, string vacío
**Errores:** La función original no detectaba emails sin TLD
**Conclusión:** Prompt válido, pero requiere revisión de seguridad

Cada interacción relevante con IA debe seguir este flujo de documentación. El registro permite trazabilidad y mejora continua.

Consejo del instructor: No necesitas documentar cada prompt trivial. Establece un umbral: documenta toda función que vaya a producción, toda lógica compleja, y todo código que haya requerido cambios significativos después de la generación. El resto puede documentarse con un simple comentario # Generado por IA - revisado [fecha].

Errores frecuentes

  • No documentar nada: El error más común. La memoria humana no es confiable para recordar qué generó la IA
  • Documentar solo los prompts exitosos: Los errores son igual de importantes para evitar repetirlos
  • Documentación excesiva: Documentar cada interacción trivial genera fatiga y se abandona rápido
  • No compartir la documentación: La documentación individual no sirve; debe ser accesible al equipo

Buenas prácticas

  • Usar un formato estándar: Plantilla uniforme para todo el equipo facilita la búsqueda y comparación
  • Documentar cerca del código: Comentarios en el código + registro externo para decisiones más complejas
  • Incluir el prompt original: Guardar el texto exacto del prompt permite reproducir y mejorar
  • Registrar también los rechazos: Saber qué no funcionó es tan valioso como saber qué funcionó
  • Revisar los registros periódicamente: Cada sprint, revisa los patrones de error para ajustar prompts y procesos

Actividad práctica

Genera con IA una función que lea un archivo CSV y calcule el promedio de una columna. Luego completa el siguiente registro de documentación:

  1. ¿Qué prompt usaste exactamente?
  2. ¿Qué cambios hiciste al código generado?
  3. ¿Qué pruebas escribiste?
  4. ¿Encontraste algún error en el código original de la IA?
  5. ¿Aceptarías este código en producción? ¿Por qué?

Usa la plantilla de registro proporcionada en esta lección.

Mini reto

Revisa el historial de tus últimas 5 interacciones con una IA generadora de código. Para cada una, completa un registro usando la plantilla de 9 pasos. Al final, responde: ¿qué patrón de error se repite? ¿Qué tipo de prompts funcionan mejor? Este análisis es el primer paso para crear una guía de prompts efectiva para tu equipo.

Resumen

  • Documentar el uso de IA proporciona trazabilidad, reproducibilidad y mejora continua
  • El registro debe incluir: requerimiento, contexto, prompts, respuesta, cambios, pruebas, errores y conclusión
  • Documentar también los rechazos es tan importante como documentar los aciertos
  • La documentación debe ser accesible al equipo, no individual
  • Revisar los registros periódicamente permite identificar patrones y mejorar el proceso
